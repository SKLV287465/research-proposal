The operating system is the lowest level of software directly interfacing with hardware and providing user applications a general, usable interface to operate upon. Essentially, it is an abstract machine and a resource manager working with \texttt{policies} to multiplex and allocate hardware resources through a simplified system call interface. \citep{Tanenbaum_Bos:mos5.pdf}. 

\subsection{Kernel}
A key component of any operating system is the operating system kernel. Although kernels are often referenced synonymously with operating systems, there is an important distinction to be made. Whilst parts of the operating system may run on less privileged modes \TODO{cite something here}, the kernel is specifically the part of the operating system which runs on the highest level of privilege. This means that the kernel has complete control and access over all computer resources in the system. Regarding this kernel, there are several designs, each having their advantages and disadvantages.

\section{Monolithic Kernel}
A monolithic kernel is a kernel design which has most if not all the operating system components inside the kernel.

\section{Microkernel}
The converse of the monolithic kernel design is a microkernel which operates on Liedtke's principle of minimality \textit{`A concept is tolerated inside the $\mu$-kernel only if moving it outside the kernel, i.e., permitting competing implementations, would prevent the implementation of the system's required functionality'} \citep{Liedtke_95}.
